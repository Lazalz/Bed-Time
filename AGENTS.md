## Code Style Guidelines

### JavaScript
- Use ES6+ features (const/let, arrow functions, template literals)
- Use descriptive variable names in camelCase
- Prefer const over let when possible
- Use array methods (map, filter, reduce) over loops
- Handle errors with try/catch blocks

### HTML
- Use semantic HTML elements
- Include proper meta tags and viewport
- Use lowercase for attributes and values
- Add alt text to images

### CSS
- Use CSS custom properties (variables) for colors/themes
- Follow BEM naming convention for classes
- Use flexbox/grid for layouts
- Include responsive breakpoints
- Add transitions for interactive elements

### Imports
- Use ES6 import/export syntax
- Group imports by type (external libraries first, then local files)
- Use relative paths for local imports

### File Structure
- Keep related files together in feature directories
- Use kebab-case for file names
- Separate assets (CSS, images) into dedicated folders
- Have fun
